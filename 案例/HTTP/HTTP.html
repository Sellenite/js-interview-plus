HTTP协议的主要特点

简单快速：每个资源URI都是固定的，在HTTP协议中处理是非常简单的

灵活：一个HTTP协议可以完成不同数据类型的传输

无连接：连接一次，就会断掉，不会保持连接

无状态：客户端和服务端是两种身份，连接断开之后，下次再连接时，服务端是无法区分这次连接是否与上一个连接是同一个身份，单从HTTP上

HTTP报文的组成部分

请求报文：请求行，请求头，空行，请求体

响应报文：响应行，响应头，空行，响应体

请求行：由请求方法字段、URL 字段 和HTTP 协议版本字段 3 个部分组成，他们之间使用空格隔开。常用的 HTTP 请求方法有 GET、POST。

请求头部：为请求报文添加了一些附加信息，由“key/value”对组成，每行一对，请求头部通知服务器有关于客户端请求的信息，典型的请求头有：如，User-Agent --- 请求的浏览器类型

空行：表示最后一个请求头部之后是一个空行，发送回车符和换行符，通知服务器以下不再有请求头部，因为头部是由多个“key/value”对组成，空行的作用表示结束。

请求包体：请求包体不在GET方法中使用，而是POST方法中使用。

POST和GET的区别

GET在浏览器回退时是无害的，回退操作实际上浏览器会从之前的缓存中拿结果，POST会再次提交请求

GET产生的地址可以被收藏，POST不可以

GET请求会被浏览器主动缓存，POST不会，除非手动设置

GET请求只能进行URI编码，POST支持多种编码

GET请求参数会被完整的保留在浏览器的历史记录里，POST的参数不会被保留

GET请求不同浏览器的长度限制不同，POST没有限制

GET参数通过URL传递，POST放在request body中

HTTP1.1 持久连接

原来的HTTP协议采用（请求-应答）模式，每个请求/应答客户端和服务器都要重新建立连接，完成之后立即断开，这就是之前说的无连接的特点

当使用keep-alive时，当对服务器有后继请求时，持久连接避免了重新建立连接

在使用持久连接的时候，如果不是管线化，传递会类似于：请求1->响应1->请求2->响应2->请求3->响应3

启动和关闭方法：HTTP头设置：Connection: Keep-Alive、Connection: Close
